{{ gettext('Ready to write your story? Here\'s how to set up a story for local development. All stories derive from the base %(story_link)s library.') | format(story_link='Story') }}

  1. {{ gettext('Fork/clone/copy the %(hipyschool_link)s or the %(hipy101_link)s repo on Github. Those a good examples of simple stories to begin with.')| format( hipyschool_link='hipyschool', hipy101_link='py101' ) }}

  2. {{ gettext('Install your local copy into a virtualenv. Assuming you have virtualenvwrapper installed, this is how you could set up your copy for local development:') }} 

    $ mkvirtualenv my_python_story
$ cd my_python_story/
$ python setup.py develop
  3. {{ gettext('Create a module for your Story. For example mystory/__init__.py.') }} {{ gettext('Define the name, title and adventures variables.') }} {{ gettext('You will have to use the gettext() function from story.translation to receive translations. More on that below.') }}
  4. {{ gettext('Create one or more Adventures. A story is a collection of one or more adventures') }}
    1. {{ gettext('Create a module for your adventure for example mystory/myfirstadventure') }}
    2. {{ gettext('Create the README.rst file describing your adventure in the previous folder') }}
    3. {{ gettext('(Optional) Make an Spanish translation. You must name it README.es.rst') }}
    4. {{ gettext('In the __init__.py file define a class called Adventure, that derives from %(baseadventure_link)s.')| format(baseadventure_link='BaseAdventure') }}

  5. {{ gettext('Since the gettext() function relies on %(pybabel_link)s, so you will have to initialize the translation by running:') | format(pybabel_link='PyBabel') }} 

    $ make msg-init
$ make msg-extract

    {{ gettext('You should see several files for each language in mystory/locale/ corresponding for the usages of the gettext() function.') }} {{ gettext('If the command fail, try again.') }}

  6. {{ gettext('(Optional) Modify and add translations in the locale files and then run:') }}

    $ make msg

  7. {{ gettext('It is recommended to create some unit tests for checking your solutions, check that your changes pass flake8, tox, and the unit tests:') }} 

    $ flake8 my_python_story tests
$ python setup.py test or py.test
$ tox

    {{ gettext('To get flake8 and tox, just pip install them into your virtualenv.') }}

  8. {{ gettext('Commit your changes and push your branch to GitHub:') }}

    $ git add .
$ git commit -m "Your detailed description of your changes."
$ git push origin
